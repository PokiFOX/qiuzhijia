# 智能体第三方用户授权接口文档

本文档描述 `/consoleapi/third-party/auth` 第三方用户授权接口，用于获取调用对话类接口所需的 JWT Token。

---

## 一、接口概览

请求地址：https://ai.fxeasy168.com:3100

| 接口类别 | 接口路径 | 方法 | 说明 |
|---------|---------|------|------|
| 用户授权 | `/consoleapi/third-party/auth` | POST | 第三方用户授权，获取 JWT Token |

---

## 二、接口详情

### 2.1 第三方用户授权

**接口**：`POST /consoleapi/third-party/auth`

**认证方式**：签名认证

**请求头**：

| 参数名 | 必填 | 说明 |
|--------|------|------|
| x-provider-code | 是 | 固定值，接入方标识码（如：`sys_job_hunting`） |
| x-signature | 是 | MD5 签名，签名算法如下 |
| x-timestamp | 是 | 当前时间戳（秒级 Unix 时间） |
| Content-Type | 是 | application/json |

**签名算法**：

```
body_md5  = MD5(JSON.stringify(body, sort_keys=true))
sign_text = "{timestamp}:{method}:{path}:{body_md5}:{secretKey}"
signature = MD5(sign_text)
```

具体步骤：

1. 将请求体按 key 字母顺序排序后 JSON 序列化
2. 计算请求体的 MD5 哈希值：`body_md5 = MD5(sorted_json_stringify(body))`
3. 拼接签名原文：`sign_text = "{timestamp}:POST:/consoleapi/third-party/auth:{body_md5}:{secretKey}"`
4. 使用 MD5 生成最终签名：`signature = MD5(sign_text)`

参数：

| 参数名    | 说明                                                         |
| --------- | ------------------------------------------------------------ |
| sign_text | 生成逻辑：{timestamp}:POST:/consoleapi/third-party/auth:{body_md5}:{secretKey} |
| body      | `/consoleapi/third-party/auth`接口的请求体                   |
| body_md5  | 对body按key字母顺序排序后计算得到的MD5哈希值                 |
| timestamp | 当前时间戳（秒级 Unix 时间）                                 |
| secretKey | 固定值，rhzn_qzj_419ce962                                    |

Python 示例：

```python
import hashlib
import json

def md5_hex(text: str) -> str:
    return hashlib.md5(text.encode("utf-8")).hexdigest()

def generate_signature(body: dict, timestamp: int, secret_key: str) -> str:
    # 注意：必须按 key 字母顺序排序
    body_str = json.dumps(body, separators=(",", ":"), ensure_ascii=False, sort_keys=True)
    # 计算请求体的 MD5 哈希值
    body_md5 = md5_hex(body_str)
    # 生成令牌文本
    sign_text = f"{timestamp}:POST:/consoleapi/third-party/auth:{body_md5}:{secret_key}"
    # 最终令牌
    return md5_hex(sign_text)

# 使用示例
body = {
    "avatarUrl": "https://example.com/avatar.jpg",
    "email": "test@example.com",
    "miniProgramOpenId": "wx_xxx",
    "nickName": "测试用户",
    "phone": "198XXXXXXXX",
    "realName": "张三",
    "unionId": "union_xxx",
    "userIdentity": "学生",
    "userSource": "求职+",
}
timestamp = int(time.time())
signature = generate_signature(body, timestamp, "your_secret_key")
```

安全规则：

- 时间戳有效期：5 分钟（300 秒）
- 签名比较使用 timing-safe 比较，防止时序攻击
- JSON 序列化时 key 必须按字母顺序排序，否则会导致签名不一致

**请求体**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| miniProgramOpenId | string | 是 | 小程序 OpenID |
| unionId | string | 否 | 微信 UnionID |
| nickName | string | 否 | 微信昵称 |
| avatarUrl | string | 否 | 微信头像 URL |
| phone | string | 是 | 手机号，优先作为用户名 |
| email | string | 否 | 邮箱 |
| realName | string | 否 | 用户真实姓名 |
| userSource | string | 是 | 用户来源，默认为 `求职+` |
| userIdentity | string | 是 | 用户身份 |

**请求示例**：

```bash
curl -X POST https://ai.fxeasy168.com:3100/consoleapi/third-party/auth \
  -H "X-Provider-Code: test_mp" \
  -H "X-Signature: a1b2c3d4e5f6..." \
  -H "X-Timestamp: 1705449600" \
  -H "Content-Type: application/json" \
  -d '{
    "avatarUrl": "https://example.com/avatar.jpg",
    "email": "XXX@163.com",
    "miniProgramOpenId": "wx_xxx",
    "nickName": "用户昵称",
    "phone": "198XXXXXXXX",
    "realName": "张三",
    "unionId": "union_xxx",
    "userIdentity": "学生",
    "userSource": "求职+"
  }'
```

**响应示例**（成功）：

```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expiresAt": 1706054400,
    "publishUrl": "",
    "user": {
      "id": "uuid...",
      "nickname": "用户昵称"
    }
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | JWT Token，有效期 7 天 |
| expiresAt | number | Token 过期时间戳（秒级） |
| publishUrl | string | 发布 URL（由调用方拼接） |
| user.id | string | 平台用户 ID |
| user.nickname | string | 用户昵称 |

**响应示例**（失败）：

```json
// 缺少请求头
{
  "success": false,
  "message": "缺少必要的请求头参数"
}

// 接入方不存在
{
  "success": false,
  "message": "接入方不存在"
}

// 接入方已禁用
{
  "success": false,
  "message": "接入方已禁用"
}

// 签名验证失败
{
  "statusCode": 401,
  "message": "签名验证失败"
}
```

**业务逻辑**：
1. 验证请求头完整性（`x-provider-code`、`x-signature`、`x-timestamp`）
2. 根据 `x-provider-code` 查询接入方，验证状态
3. 使用 `secretKey` 验证 MD5 签名和时间戳（JSON 按 key 字母顺序排序后计算）
4. 根据 `miniProgramOpenId` 查询第三方用户：
   - **不存在**：创建平台用户 + 创建第三方用户关联
     - `username` 优先使用`RH${date}${phone.slice(-4)}`，否则使用 `tp_{code}_{openid后8位}`
     - `phone`、`email`、`realName` 保存到 `user` 表
     - `userSource`、`userIdentity` 保存到 `third_party_user` 表
   - **存在**：更新用户信息（昵称/头像/来源/身份/手机号/邮箱/真实姓名）
5. 生成 JWT Token（有效期 7 天）
6. 返回 Token 和用户信息

---

## 三、错误码说明

| 状态码 | 错误信息 | 说明 |
|--------|---------|------|
| 401 | 签名验证失败 | 签名错误或过期 |
| 403 | 第三方用户已被禁用 | 用户状态异常 |
| 403 | 关联的平台用户已被禁用 | 平台用户状态异常 |
